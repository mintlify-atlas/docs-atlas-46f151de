### Reporting Issues

Found a bug or have a feature request?

<Steps>
  <Step title="Search Existing Issues">
    Check if your issue has already been reported in the relevant repository's Issues tab.
  </Step>
  
  <Step title="Create a New Issue">
    If your issue is new, create a detailed issue report including:
    - Clear description of the problem
    - Steps to reproduce
    - Expected vs actual behavior
    - Version information
    - Code examples (if applicable)
  </Step>
  
  <Step title="Follow Up">
    Monitor your issue for responses from maintainers and provide additional information if requested.
  </Step>
</Steps>

## FAQ and Common Issues

Before reaching out for help, check if your question is answered in these resources:

### Installation Issues

<Accordion title="Leo or SnarkOS won't install">
  - Ensure Rust 1.79+ is installed: `rustc --version`
  - Update Rust: `rustup update`
  - Make sure Git submodules are initialized: `git submodule update --init --recursive`
  - Check the [Installation Guide](/installation) for platform-specific instructions
</Accordion>

<Accordion title="Command not found after installation">
  Add Cargo's bin directory to your PATH:
  
  ```bash
  export PATH="$HOME/.cargo/bin:$PATH"
  ```
  
  Add this line to your `~/.bashrc` or `~/.zshrc` and restart your terminal.
</Accordion>

### Development Issues

<Accordion title="Program won't compile">
  - Check for syntax errors in your Leo code
  - Ensure all imports are correct
  - Verify your program name matches in both `src/main.leo` and `program.json`
  - Try `leo clean` followed by `leo build`
</Accordion>

<Accordion title="Deployment fails">
  - Ensure you have sufficient testnet credits
  - Verify your private key is correct
  - Check that your program name is unique (not already deployed)
  - Program names should be at least 10 characters to minimize fees
</Accordion>

<Accordion title="Can't see my transaction">
  - Transactions may take a few minutes to be included in a block
  - Check the transaction ID in a block explorer:
    - [Provable Explorer](https://testnet.explorer.provable.com/)
    - [Aleoscan](https://testnet.aleoscan.io/)
  - Ensure you're checking the correct network (testnet vs mainnet)
</Accordion>

## Tips for Getting Better Help

When asking for help, provide:

<Steps>
  <Step title="Context">
    Explain what you're trying to accomplish and why.
  </Step>
  
  <Step title="Code Examples">
    Share relevant code snippets or links to your repository.
  </Step>
  
  <Step title="Error Messages">
    Include complete error messages and stack traces.
  </Step>
  
  <Step title="Environment Details">
    Share your operating system, tool versions, and any relevant configuration.
  </Step>
  
  <Step title="What You've Tried">
    Describe steps you've already taken to solve the problem.
  </Step>
</Steps>

<Tip>
The more context and details you provide, the faster the community can help you!
</Tip>
